ES6

ES6 is the current version of JavaScript that is standardized across the web. There is only ever one version of JavaScript in release at any given time. ES6 comes with new features and also more succinct patterns of writing JavaScript, which we will examine now.

Default Parameters

Sometimes, when you are writing code, you want to have a default parameter set for incoming arguments. Say, for example, you are writing a function that multiplies any two numbers together to create an even number. By default, you want the second number to be two. You can either check each time to see if the second number inputted is two like so: 

function makeEven(number1, number2) {
  if(number2==2) {
    return number1 * number2;
  } else {
    break;
  };
}
```+.                             

Or, you can use the faster ES6 style of writing a default parameter into your function, like so: 

```js
function makeEven(number1, number2=2) {
  return number1 * number2;
}

Template Literals

Template literals help you write string concatenation faster, and also allows you to input values into strings. Before, to concatenate a string, you would have to write it like so: 

let name = "Marsha";
let age = 23;
let major = "Computer Science";

let sentence = name + " is " + age + " years old and is studying " + major + ".";

console.log(sentence)

>>Marsha is 23 years old and is studying Computer Science.

Now, with string template literals, you can simply add a ${} sign with whatever value or variable you want inside of it, and remove the plus signs.

Note You must wrap the entire string in backticks, ``, in order for string template literals to work. 

let name = "Marsha";
let age = 23;
let major = "Computer Science";

let sentence = `${name} is ${age} years old and is studying ${major}.`

console.log(sentence); 

>>Marsha is 23 years old and is studying Computer Science.

You can also input expressions (computations) into template literals. 

function add(num1, num2) {
  console.log(`The sum is ${num1 + num2}.`);
}

add(3, 4)
>>The sum is 7.

Destructuring

Destructuring is a fast and convenient way to extract data from objects and arrays and put them into variables.

Destructuring an array:

const numbers = [1, 2, 3, 4, 5];
[a, b, ...rest] = numbers
console.log(a, b, rest)
>>1 2 [3, 4, 5];

Destructuring an object: 

const student = { major: "Political Science" };
const {major: major} = student; 
console.log(major)
>>Political Science

Using destructuring to declare variables: 

let a;
let b;
[a, b] = [10, 20]

Destructuring in a for-of loop: 

const students = ['Jose', 'Keanu'];

for (const [index, element] of students.entries()) {
  console.log(index, element)
}
// Output:
// 0 "Jose"
// 1 "Keanu"

Spread and Rest

Spread "expands" an array into individual elements. You can use it to add items to arrays, combine arrays or objects, or spread out an array to be used as individual function arguments.

Concatenating arrays:

const nums1 = [3, 4, 5];
const nums2 = [0, 1, 2];

const nums3 = [...nums2, ...nums1];

console.log(nums3)
//output 
//[0, 1, 2, 3, 4, 5]

Copying an array: 

const arr = [0, 1, 2, 3, 4, 5]
const copiedArr = [...arr]
console.log(copiedArr);
//output
//[0, 1, 2, 3, 4, 5]

Using spread with arguments in a function:

const students = ["Oliver" , "Clara", "Mathias"];
const registerStudents = (s1, s2, s3) => { console.log(`${s1}, ${s2}, and ${s3} are all registered for CS 100.`) } 

registerStudents(...students)
//output
//Oliver, Clara, and Mathias are all registered for CS 100.

Rest takes remaining individual elements of an array or objects and collects them into a single variable, called 'rest'.

Using rest to collect items in an array:

let nums = [1, 2, 3, 4, 5, 6, 7, 8, 9];
let [a, b, ...rest] = nums
console.log(a, b, rest)
//output
//1 2 [3, 4, 5, 6, 7, 8, 9]

Using rest as an argument in a fucntion:

function add(...nums) {
  let result = 0;
  for(let num of nums) result += num;
  return result
}

add(1)
//output
//1

add(1, 2)
//output
//3

add(1, 2, 3, 4, 5)
//output
//15

Scoping with let and const

The let statement creates a block-scoped variable. The variable exists only inside the block, and disappears once the block has been run.

Block scope with let: 

let x = 1; 
  if(x === 1) {
    let x = 10;
  console.log(x);
   }
console.log(x);
//output
//10
//1

The const statement also declares a variable inside the block scope like let. The value of const can't be changed through re-assignment. Also, any values declared with const cannot be redeclared.

Block scope with const: 

const num = 7;
if(num ===7) {
  const num = 10;
  console.log(num);
 }
console.log(num);
//output
//10
//7

Modules

Modules are a way to break up large JavaScript programs to import and export functionality as needed. Modules are incredibly useful to developers, as the size and complexity of JavaScript has grown a great deal since its inception.

To export a function as a module, simply put the export keyword in front of the function keyword

export function sayHello() {
  return 'Hello, World!'
}

To import modules, use the 'import' keyword in the file you wish to use the module in, along with the filepath to where the module is stored. 

import {sayHello} from './hello.js'

Closures

A closure in JavaScript gives you access to a function's outer scope within its inner scope; that is to say, anything in the outer scope of a function is also available in its inner scope.

Here is an example of a nested function using closure to access variables in its outer function. 

function sayHello() {
  var name = "Mark" // this variable is in the outer scope of the function
    function greet() {
      alert(`Oh, hi ${name}.`) // You can access the variable 'name' in the outer scope in the inner scope
    }
  greet()
}
//Output
//Oh, hi Mark.
